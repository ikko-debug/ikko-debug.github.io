Hymba：小型语言模型的新标杆——融合SSM与Attention的混合架构

随着Transformer在自然语言处理（NLP）中的统治地位，其高计算复杂度和内存需求的短板也日益凸显。近年来，状态空间模型（SSM，如Mamba）以线性复杂度崭露头角，但其回忆能力不足。2025年ICLR上，NVIDIA团队提出了Hymba，一种小型语言模型（LM），通过创新的“混合头”（Hybrid-Head）架构，巧妙融合SSM和Attention，实现了性能与效率的双赢。本文将深入剖析SSM的结构、Hymba的设计，以及它如何在同一层内并行融合两者。

一、SSM（状态空间模型）的结构

SSM是一种基于控制论的序列建模方法，Mamba（Gu & Dao, 2023）是其代表。它通过线性复杂度处理长序列，成为Transformer的有力竞争者。以下是SSM的核心结构：

1.1 SSM的基本原理

SSM将序列建模视为一个动态系统，用状态转移方程描述输入到输出的映射：

  • 输入：序列( X = [x_1, x_2, …, x_n] )，每个( x_t \in \mathbb{R}^d )。
  • 状态：隐状态( h_t \in \mathbb{R}^s )（s是状态维度，如Hymba中的16）。
  • 输出：( y_t \in \mathbb{R}^d )。

其离散形式为：
[
h_t = A h_{t-1} + B x_t
]
[
y_t = C h_t
]

  • ( A \in \mathbb{R}^{s \times s} )：状态转移矩阵。
  • ( B \in \mathbb{R}^{s \times d} )：输入投影矩阵。
  • ( C \in \mathbb{R}^{d \times s} )：输出投影矩阵。

1.2 Mamba的改进

Mamba在传统SSM基础上引入了数据依赖的参数和门控机制：

  • 动态参数：
    • ( A )：固定但可学习。
    • ( B = W_B x_t )：随输入动态变化。
    • ( C = W_C x_t )：同上。
    • ( \Delta = \text{Softplus}(W_{\Delta} x_t) )：时间步长，控制遗忘速度。
  • 门控：引入门控向量( G = W^G x_t )，输出为：
    [
    y_t = G \odot (C h_t)
    ]
  • 计算：
    • 递归形式：( h_t = \exp(A \Delta_t) h_{t-1} + B_t \Delta_t x_t )。
    • 并行形式（推理优化）：用卷积或矩阵运算展开：
      [
      Y = G \odot \alpha(A, B, C, \Delta) X
      ]
      其中( \alpha )是预计算的转移函数。

1.3 SSM的优劣

  • 优点：线性复杂度（O(n)），小缓存（Hymba中仅1.87MB）。
  • 缺点：低分辨率回忆，推理能力弱（Tab. 1显示Mamba在回忆任务仅19.23%）。

二、Hymba的结构

Hymba-1.5B是Hymba家族的主力型号，参数量1.52亿，专为小型设备设计。其架构以32层混合头模块为核心，融合了SSM和Attention，并引入优化策略。

2.1 总体架构

  • 层数：32层。
  • 隐藏维度：1600。
  • 输入：原始序列( X )前置128个Meta Tokens，形成( \tilde{X} = [R, X] )。
  • 输出：通过线性投影生成词汇概率。

参数配置

属性 Hymba-1.5B
层数 32
隐藏维度 1600
SSM状态维度 16
Attention头数 25
查询分组（GQA） 5
全局注意力层 3（1、16、32）
窗口大小 1024
MLP隐藏维度 5504

2.2 混合头模块（Hybrid-Head Module）

每层是一个混合头模块，同时包含Attention和SSM头，输入( \tilde{X} )被并行处理。

Attention头

  • 数量：25个头。
  • 计算：
    [
    Y_{\text{attn}} = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right) V
    ]
    • ( Q = W^Q \tilde{X}, K = W^K \tilde{X}, V = W^V \tilde{X} )。
    • 头维度( d = 1600 / 25 = 64 )。
  • 类型：
    • 全局（3层）：完整序列。
    • 局部（SWA，29层）：窗口1024。
  • GQA：25头分5组，共享键值。

SSM头

  • 实现：基于Mamba。
  • 计算：
    [
    Y_{\text{ssm}} = G \odot \alpha(A, B, C, \Delta) W^{SSM} \tilde{X}
    ]
    • ( G = W^G \tilde{X} )：门控。
    • ( X_{ssm} = W^{SSM} \tilde{X} )：输入特征。
    • ( A, B, C, \Delta )同Mamba。
  • 状态维度：16。

融合

  • 输出融合：
    [
    Y = W_{\text{out.proj}} \left( \beta_1 \text{norm}(Y_{\text{attn}}) + \beta_2 \text{norm}(Y_{\text{ssm}}) \right)
    ]
    • ( \beta_1, \beta_2 )：可学习缩放。
    • ( \text{norm} )：平衡幅度差异（Fig. 9）。

2.3 Meta Tokens

  • 数量：128个( R = [r_1, …, r_{128}] )。
  • 作用：优化注意力分布，存储知识。
  • 实现：训练时优化，推理时预计算为KV初始化。

2.4 KV优化

  • 全局+局部：3层全局，29层SWA。
  • 跨层共享：32层分13组，每组共享KV。

三、Hymba如何融合SSM与Attention

Hymba的融合方式是其最大亮点，区别于Jamba的串联设计，它采用并行融合：

3.1 并行处理

  • 同一层内：输入( \tilde{X} )同时送入Attention和SSM头。
  • 协同作用：
    • Attention：提供“快照记忆”，高分辨率回忆（Tab. 1 recall从19.23%提升至51.79%）。
    • SSM：提供“渐退记忆”，高效总结上下文。
  • 对比串联：Jamba分层交替可能导致瓶颈，Hymba并行避免信息丢失（ERF更大，Fig. 8）。

3.2 输出融合

  • 加权平均：Attention和SSM输出通过归一化和缩放融合，确保稳定性。
  • 实验验证：Tab. 11显示并行优于拼接（concat），性能更高且参数更少。

3.3 Meta Tokens的桥梁作用

  • 注意力引导：减少BOS关注（Fig. 11），让Attention更专注关键token。
  • SSM增强：降低熵（Fig. 13），提升SSM的上下文提取能力。

3.4 KV优化的协同

  • SSM支持：全局上下文由SSM总结，允许Attention用局部SWA。
  • 结果：缓存减少11.67倍，吞吐量提升3.49倍（Tab. 2）。

四、Hymba的性能与意义

  • 性能：Hymba-1.5B平均准确率61.31%，超Llama-3.2-3B（Tab. 2）。
  • 效率：缓存134MB，吞吐量840 token/s。
  • 意义：为小型LM树立标杆，适合边缘设备，推动混合架构研究。

五、总结

Hymba通过混合头模块，在每层并行融合SSM和Attention，结合Meta Tokens和KV优化，实现了小型LM的突破。它不仅是对Transformer效率优化的新尝试，也是SSM与Attention协同的典范。想试试Hymba？Hugging Face上已有模型（Hymba-1.5B-Base/Instruct），值得一探究竟！
